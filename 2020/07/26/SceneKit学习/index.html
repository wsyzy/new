Scene Kit_1 基础

1.什么是物理引擎

​ 物理引擎通过为刚性物体赋予真实的物理属性的方式来计算运动、旋转和碰撞反映。创建3D场景并添加至应用程序,提供基础的物理模拟,粒子效果,定义了虚拟的三维世界的自然规律.SceneKit(SK)是WWDC12推出的OS X平台的Cocos 3D渲染引擎框架。支持粒子效果，物理模拟，脚本事件，多程渲染，支持iOS平台。

SCNView

在macOS中，SCNView是NSView的子类，在iOS和tvOS中，SCNView是UIView的子类。SCNView用于显示SceneKit的3D场景，而需要设置场景的相关内容和属性需要通过SCNScene。SCNView需要遵循SCNSceneRenderer协议和SCNTechniqueSupport协议。

  • allowsCameraControl

    ​ 允许操纵相机，用户可以改变视角的位置和方向

    ​ SCNView

  • showsStatistics

    ​ 一个布尔值，确定SceneKit是否在附件视图中显示渲染性能统计信息。

    ​ SceneKit统计信息视图显示有关场景渲染性能和GPU资源使用情况的各种信息，包括每秒帧数（fps）计数器。在macOS中，单击统计视图中的齿轮按钮以显示一个面板，其中包含用于调整SceneKit场景渲染的其他控件。

  • SceneKit的坐标系

    ​ SceneKit使用右手坐标系，其中（默认情况下）视图的方向沿着负z轴，如果需要与其他3D框架共享数据，先了解其框架是右手坐标系还是左手坐标系。其实也很好转化，就是Z 轴的正负不一样而已。

SCNScene

SCNScene对象表示三维场景及其内容,添加各种元素到场景,读取场景文件,将场景写入文件等。节点组成的层级结构（树形结构）构成整个场景的骨架，相机、灯光、几何体等“内容”附着在节点上，这些节点以及附着在节点上属性，构成了一个可见的3D场景。通俗说是游戏场景.

SCNNode

在SCeneKit 中节点是一个抽象概念,节点是看不见摸不着的东西,没有几何形状,但是有位置,及其自身坐标.在场景中创建一个节点后,就可以在这个节点上放游戏元素了.比如任务模型,灯光,摄像机等.节点上可以添加子节点,每个SCNScene提供一个根结点作为层级结构的顶层

  • name属性:与节点关联的名称,以使管理场景图更容易。从场景文件加载的节点可能具有艺术家使用三维创作工具指定的名称。使用childNode(withName:递归：）或childNodes（passingTest:）等方法时需要按名称从场景图中检索节点

1. SCNLight

用于添加光源，连接到一个节点照亮现场，可以给场景添加不同的灯光，模拟逼真的环境

  • 境光(Ambient light)：环境光源是控制场景整体亮度的源头。

  • 位光(Omni light)：白天天空中的明亮的光芒 - 太阳 - 基本上是全能的光,向四周提供亮光

  • 向光(Directional light)：这种类型在特定方向发射平行光线。

  • 源(Sopt light)：这种类型的光也是定向的，但是光从空间中的单个点发射并且以锥形展开。

    SCNLight

    2. SCNVector3

三分失向量的一种表示.SceneKit用于多种用途，例如描述节点或顶点位置、曲面法线以及缩放或平移转换。应根据使用矢量的上下文来解释不同的矢量组件。SceneKit中的坐标以m为单位

3. SCNCamera

一组摄像机属性，可以连接到一个节点，以提供显示场景的角度。

若要显示场景，必须将其“摄影机”属性包含摄影机对象的节点指定为视点。 包含摄影机的SCNNode对象定义了一个视点，即摄影机的位置和方向。摄影机的视图方向始终沿节点局部坐标系的负z轴。若要将摄影机指向场景的不同部分，请使用包含摄影机的节点的“位置”、“旋转”或“变换”属性。（或者，要确保摄影机始终指向场景中的特定元素，即使该元素移动，请将SCNLookAtConstraint对象附加到包含摄影机的节点。） SCNCamera对象本身定义了从其角度看渲染场景的形状和部分外观。默认情况下，相机定义透视投影，使用“调整相机透视”中列出的属性（如下图所示）可以控制透视投影的视野（FOV）和远近可见性限制。

4. SCNGeometry

  • SCNBox中的chamferRadius为可动画的倒角半径

最大拐角半径是盒子最小尺寸的一半。例如，如果一个盒子的宽度和长度均为5.0，高度为10.0，则其最大角半径为2.5。在这些尺寸下，盒子的四个圆形垂直边缘连接在一起形成一个圆柱体，并且其垂直面消失了。将拐角半径增加到最大值以上不会产生可见效果

  • geometry.materials.first?.diffuse.contents = color 表示这对几何图形的内容进行赋值
  • plane.materials.first?.isDoubleSided = true

    ​ 当此属性的值为false(默认值)时，SceneKit将使用来自一侧的材料呈现表面。几何中的顶点数据和法线向量指定每个多边形的哪一侧被认为是它的正面，而该几何图形相对于摄像机的方向决定了当前可见的前表面。如果将此属性的值更改为“是”，则SceneKit将呈现每个多边形的前表面和后表面。SceneKit假设曲面法线是正面法线向量的负值。可以对此属性值进行动画更改。

[SceneKit] 不会 Unity3D 的另一种选择

5. SCNPhysicsBody

  • 附加到场景图节点的物理模拟属性.

  • 当SceneKit准备渲染新框架时，它将对附着到场景节点上的物理实体执行物理计算。这些计算包括重力，摩擦以及与其他物体的碰撞。您还可以对body施加自己的力量和冲力。SceneKit完成这些计算后，它将在渲染框架之前更新节点对象的位置和方向。

  • 要将物理学添加到节点，请创建并配置一个对象，然后将其分配给该对象的属性。在向节点对象施加力或脉冲之前，它必须与它关联

  • tyoe:

    ​ 确定模拟中物体与力和其他物体的交互方式。静态物体不受力和碰撞的影响，并且不能移动。动态物体会受力以及与其他物体碰撞的影响。运动(kinetic)物体不受力或碰撞的影响，但是直接移动运动物体会导致碰撞，从而影响动态物体。

  • physicalShape

    ​ 碰撞检测目的的三维实体形式,使用简单的形状而不是节点可见几何的精细细节时，物理模拟的运行速度更快.当SceneKit 对场景中的对象执行接触检测和其他模拟时，它将使用物理形状而不是可见对象的渲染几何形状。这种方法可以提高模拟性能，

  • mass属性

    body的质量,以千克为单位.物体的质量会影响其动量以及其对力的反应方式。动态物体的默认质量为1.00。静态和运动物体的默认质量为0.00，但是这些物体不受质量的影响。

SceneKit学习_2 捕获几何体

为结点添加物理效果

  • categoryBitMask

    ​ 分类位掩码,定义了这个物体所属的类别.

    ​ 您可以将场景中的每个节点分配给一个或多个类别，其中每个类别对应于位掩码中的一位。您可以定义应用中使用的掩码值。当SceneKit渲染场景时，它使用逐位AND运算将每个节点的属性与参与渲染过程的每个其他对象的类别位掩码（灯光，照相机和技术）进行比较。如果结果为非零值，则SceneKit会在渲染时包括该节点。默认类别位掩码为1。

  • collisionBitMask

    ​ 一个定义哪个类型的physics body可以和这个类型的physics body碰撞的掩码(mask)

    ​ 当两个物体相互接触时就可能产生碰撞.SceneKit通过按位与运算将该body的collision mask和其他body的category mask进行比较.如果结果非零值,那么这个body就会受碰撞影响.每个body都需要独立地选择是否收其他的body碰撞影响.例如，您可能选择避免碰撞计算，这将使物体的速度变化可以忽略不计。

    ​ 默认值为all（一个位掩码，其每一位都被启用），该body将与所有其他类别的body产生碰撞。

  • contactTestBitMask

    ​ 接触测试位掩码.当两个物理物体重叠时，可能会发生接触。

    ​ SceneKit通过执行按位与运算，将body的contact mask与其他body的category mask进行比较。如果结果为非零值，SceneKit将创建一个SCNphysicsContact的对象，并将消息发送到物理场景世界中的contactDelegate对象。为了获得最佳性能，请仅在你想要交互的contact mask中设置位.在OS X v10.11或iOS 9或更高版本上运行的应用程序，此属性的值默认为零，并且不需要与碰撞掩码(collisionBitMask)匹配-也就是说，只要两个body相交，这一对body就会生成联系消息，无论它们是否碰撞或只是穿过彼此
