Xcode中的new group、new group without folder与New Group from Section的区别

![屏幕快照 2020-06-25 16.43.53](/Users/qaq/Desktop/屏幕快照 2020-06-25 16.43.53.png)

New Group

不能直接重命名,会在项目文件夹中生成New Group文件夹![image-20200625164903040](/Users/qaq/Library/Application Support/typora-user-images/image-20200625164903040.png)

![屏幕快照 2020-06-25 16.50.41](/Users/qaq/Desktop/屏幕快照 2020-06-25 16.50.41.png)

New Group Without Folder

可以直接重命名,不会在项目文件夹中生成新的文件夹

New Group from Section

新建一个group并将选中的文件归到里面去,可以重新命名

Mac上搭建基于hexo的个人博客

环境配置

git和node.js

git : 开源分布式版本控制系统
Node.js : 运行在服务端的 JavaScript

git

mac上可以直接用Homebrew装git, 当然 得先装Homebrew
ssh私/公匙 github上配置公匙

node.js

用nvm安装

安装hexo

使用Node.js 中的npm包管理工具,将hexo这个模块装到电脑中

最后

分别用v来检查是否安装成功
权限问题试试 sudo 或者忽略安全方面的警告强制安装

使用Hexo(查看官方文档)

创建你的blog文件 , 也称 建站

切入你准备用来搭建blog的目录
先执行 hexo init
再执行 npm install

预览本地博客

仍在你的blog目录下
执行 hexo start
浏览器打开 http://localhost:4000/ 预览, ctr+c 停止运行

部署到github上

github上的操作

github上新建一个 repository, 注意命名的固定模式为: userName.github.io
配置公匙在前面说了, 检查一下

本地配置

配置github的用户名和密码,命令行输入

配置blog文件中的_config.yml文件中的 #Deployment 部分,如下. 注意缩进以及github网址格式 保持一致, 修改完后保存退出

hexo部署

在blog目录下运行 hexo g -d
访问 https://用户名.github.io 就可以看到你的远程博客了

更多设置

网站基本配置 #Site

打开你的blog的 _config.yml 文件, 按个人情况填写 #Site 部分配置 , language填写 zh-Hans 表示使用简体中文

个人头像
添加标签

about、categories、tag页面

添加Read More按钮

新建文章

git 的 config 要配置用户名和邮箱

注意引号和空格,配置成功会有输入密码(github)提示
用户名:wsyzy 而不是Wendyzy
